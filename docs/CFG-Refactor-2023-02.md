## CFG重构设计思路

### 思路

 点击文件树函数节点时，只加载一次CFG图，所有影响CFG覆盖操作都只针对测试用例数据产生变化，测试用例的变化，只会影响节点和边的颜色，通过d3设置节点和边的颜色，
更新节点和边的测试用例（testId）数据，不重新加载整个CFG图，保留整个CFG图的之前的操作情况


### 影响CFG覆盖操作
1. 删除单个测试用例 
2. 清空整个函数的测试用例 
3. 添加测试用例
4. 修改测试用例
5. 局部一键执行测试用例
6. 文件树区域一键执行测试用例
